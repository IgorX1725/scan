visão global

O NAPS2, além da GUI primária, também oferece uma interface de linha de comando (CLI) por meio do programa NAPS2.Console.exe. Isso permite que os documentos digitalizados e salvos sejam automatizados e / ou com script.

Uso

O NAPS2.Console usa opções padrão de estilo unix. Cada opção tem um formato longo (por exemplo, "--output"), e alguns têm um formato curto (por exemplo, "-o"). Algumas opções são sinalizadores e algumas devem receber um valor.

Opções Requeridas

Pelo menos uma dessas opções deve ser especificada.

-o, --output {caminho}
Especifica o caminho, o nome e a extensão do arquivo de saída a ser salvo. A extensão determina o tipo de saída (por exemplo, ".pdf" para um arquivo PDF, ".jpg" para uma imagem JPEG, etc.). 

naps2.console -o "F: \ recibo digitalizado.pdf"

-e, --email {nome_do_arquivo}
Especifica que um e-mail deve ser enviado com a verificação anexada com o nome e extensão de arquivo fornecidos. A extensão determina o tipo de anexo (por exemplo, ".pdf" para um arquivo PDF, ".jpg" para uma imagem JPEG, etc.). 

naps2.console -e "scan.pdf"

-a, --autosave
Especifica que as configurações de salvamento automático do perfil selecionado devem ser usadas. O perfil deve ter Auto Salvar ativado. 

naps2.console -a

Opções de Importação

-i, --import {filenames}
Especifica o nome e o caminho de um ou mais arquivos pdf / image para importar. Os arquivos importados são prefixados na saída na ordem em que são especificados. Vários arquivos são separados por um ponto-e-vírgula (";"). 

naps2.console -i "F: \ scan.pdf incompleto" -o "F: \ complete scan.pdf"

--importpassword {password}
Especifica a senha a ser usada para importar PDFs criptografados. 

naps2.console -i "F: \ encrypted.pdf" --importpassword "123456" -o "F: \ complete scan.pdf"

Combinando e convertendo arquivos

A opção de importação -i / - pode ser usada em combinação com a opção "-n 0" para combinar vários arquivos juntos ou para alterar o tipo de um arquivo sem fazer nenhuma varredura.

Para combinar várias imagens em um PDF:

naps2.console -i "F: \ a.jpg; F: \ b.jpg; F: \ c.jpg" -n 0 -o "F: \ abc.pdf"

Para combinar vários PDFs juntos:

naps2.console -i "F: \ a.pdf; F: \ b.pdf; F: \ c.pdf" -n0 -o "F: \ abc.pdf"

Para converter um TIFF em um PDF:

naps2.console -i "F: \ a.tiff" -n 0 -o "F: \ a.pdf"

Para converter entre tipos de imagem:

naps2.console -i "F: \ a.png" -n 0 -o "F: \ a.jpg"

Cortando arquivos importados

Para importar apenas parte de um arquivo PDF ou TIFF, você pode usar a notação de fatia . Nota: Com esta notação, os números das páginas começam em 0!

Para obter a primeira página:

naps2.console -i "F: \ abcd.pdf [0]" -n 0 -o "F: \ a.pdf"
Para substituir a terceira página:
naps2.console -i "F: \ abcde.pdf [: 2]; F: \ r.pdf; F: \ abcde.pdf [3:]" -n 0 -o "F: \ abrde.pdf"

Primeira página: [0]
Segunda página: [1]
Terceira página: [2]
Última página: [-1]
Segunda e última página: [-2]
Primeiras duas páginas: [:2]
Pule as duas primeiras páginas: [2:]
Últimas duas páginas: [-2:]
Pule as duas últimas páginas: [:-2]
Pule as primeiras / últimas duas páginas: [2:-2]

Opções de PDF

Essas opções são aplicáveis ​​apenas se o arquivo de saída for um PDF.

--pdftitle {title}
Especifica o título dos metadados gerados em PDF.

--pdfauthor {author}
Especifica o autor dos metadados gerados em PDF.

--pdfsubject {subject}
Especifica o assunto dos metadados gerados em PDF.

--pdfkeywords {keywords}
Especifica as palavras-chave para os metadados PDF gerados. 

naps2.console -o "F: \ scan.pdf" --pdftitle "Recibos de Janeiro" --pdfauthor "John Smith" --pdfsubject "Recibos" --pdfkeywords "Janeiro, Recibos"

--usesavedmetadata
Especifica que os metadados configurados na GUI devem ser usados ​​para PDF gerado. 

naps2.console -o "F: \ scan.pdf" --usesavedmetadata

--encryptconfig {caminho}
Especifica o nome e o caminho de um arquivo XML para configurar a criptografia para o PDF gerado.

naps2.console -o "F: \ encrypted.pdf" --encryptconfig "F: \ pdfencryption.xml"
pdfencryption.xml: 
<? xml version = "1.0">
<PdfEncryption xmlns: xsi = "http://www.w3.org/2001/XMLSchema-instance" xmlns: xsd = "http://www.w3.org/2001/XMLSchema">
  <EncryptPdf> true </ EncryptPdf>
  <UserPassword> 123 </ UserPassword>
  <OwnerPassword> 456 </ OwnerPassword>
  <AllowContentCopyingForAccessibility> false </ AllowContentCopyingForAccessibility>
  <AllowAnnotations> false </ AllowAnnotations>
  <AllowDocumentAssembly> false </ AllowDocumentAssembly>
  <AllowContentCopying> false </ AllowContentCopying>
  <AllowFormFilling> false </ AllowFormFilling>
  <AllowFullQualityPrinting> false </ AllowFullQualityPrinting>
  <AllowDocumentModification> false </ AllowDocumentModification>
  <AllowPrinting> false </ AllowPrinting>
</ PdfEncryption>

--usesavedencryptconfig
Especifica que a criptografia configurada na GUI deve ser usada para o PDF gerado. 

naps2.console -o "F: \ encrypted.pdf" --usesavedencryptconfig

--pdfcompat {type}
Especifica o padrão usado para o PDF gerado ("padrão", "A1-b", "A2-b", "A3-b" ou "A3-u"). 

naps2.console -o "F: \ pdfa.pdf" --pdfcompat A1-b

Opções de imagem

--jpegquality {#}
(Padrão: 75) Especifica a qualidade dos arquivos JPEG salvos (0-100). 

naps2.console -o "F: \ scan.jpg" --jpegquality 90

--tiffcomp {type}
Especifica o tipo de compactação usado para arquivos TIFF ("auto", "lzw", "ccitt4" ou "none"). 

naps2.console -o "F: \ scan.tiff" --tiffcomp nenhum

Opções de email

Essas opções são aplicáveis ​​somente se -e / - email for especificado.

--to {address}
Especifica um ou mais endereços (separados por vírgula) dos destinatários do email.

--cc {address}
Especifica um ou mais endereços (separados por vírgula) dos destinatários do email.

--bcc {address}
Especifica um ou mais endereços (separados por vírgula) dos destinatários do email.

--subject {subject}
Especifica o assunto da mensagem de email.

--body {text}
Especifica o corpo do texto da mensagem de email.

--autosend
Especifica que o email deve ser enviado sem solicitar que o usuário o edite primeiro. Observe que o Outlook pode não permitir que o email seja enviado sem a aprovação do usuário.

--silentsend
Requer --autosend. Especifica que o usuário não deve ser solicitado a autenticar. Isso pode resultar em um erro se a autenticação for necessária. Observe que o Outlook pode não permitir que o email seja enviado sem a aprovação do usuário. 

naps2.console -e "scan.pdf" --to "jsmith@example.com" --subject "Documento digitalizado" --body "Aqui está." --autosend --silentsend

Opções de OCR

Se as opções de OCR não forem especificadas, as opções da GUI serão usadas.

--ocrlang {code}
Especifica o código de três letras para o idioma usado para o OCR (por exemplo, 'eng' para inglês, 'fra' para francês, etc.). Vários códigos podem ser separados pelo símbolo '+'. Implica --enableocr. Referência: http://www.loc.gov/standards/iso639-2/php/code_list.php 

naps2.console -o "F: \ scan.pdf" --ocrlang "eng"

--enableocr
Especifica que o OCR deve ser usado ao gerar PDFs. 

naps2.console -o "F: \ scan.pdf" --enableocr

--disableocr
Especifica que o OCR NÃO deve ser usado ao gerar PDFs. Substitui --enableocr. 

naps2.console -o "F: \ scan.pdf" --disábil

Opções de encomenda

--interleave
Especifica que as páginas devem ser intercaladas. Por exemplo, se você digitalizar páginas na ordem "1, 3, 5, 2, 4, 6", essa opção resultará na ordem "1, 2, 3, 4, 5, 6".

--altinterleave
Especifica que as páginas devem ser intercaladas de maneira alternativa. Por exemplo, se você digitalizar páginas na ordem "1, 3, 5, 6, 4, 2", essa opção resultará na ordem "1, 2, 3, 4, 5, 6".

--deinterleave
Especifica que as páginas devem ser desintercaladas. Por exemplo, se você tiver páginas na ordem "1, 4, 2, 5, 3, 6", usar esta opção resultará na ordem "1, 2, 3, 4, 5, 6".

--altdeinterleave
Especifica que as páginas devem ser desintercaladas de uma maneira alternativa. Por exemplo, se você tiver páginas na ordem "1, 6, 2, 5, 3, 4", essa opção resultará na ordem "1, 2, 3, 4, 5, 6".

--reverse
Especifica que as páginas devem ser invertidas. Por exemplo, se você digitalizar páginas na ordem "6, 5, 4, 3, 2, 1", essa opção resultará na ordem "1, 2, 3, 4, 5, 6".

Opções divididas

Essas opções são mutuamente exclusivas.

--split
Especifica que cada página deve ser salva em seu próprio arquivo PDF ou TIFF.

--splitscans
Especifica que as páginas de cada varredura devem ser salvas em seu próprio arquivo. Isso só faz sentido com a opção -n.

--splitpatcht
Especifica que as páginas devem ser salvas em vários arquivos, separadas por páginas do Patch-T .

--splitsize {#}
Especifica que as páginas devem ser salvas em vários arquivos, com o número de páginas por arquivo.

Outras opções

-p, --profile {name}
Especifica o nome do perfil a ser usado durante a varredura. Perfis são definidos usando a GUI. Se essa opção não for especificada, o perfil usado mais recentemente na GUI será selecionado. 

naps2.console -o "F: \ recibo digitalizado.pdf" -p "Canon MP495 (cor)"

-v, --verbose
Exibe informações de progresso. Se não for especificado, nenhuma saída será exibida após o sucesso. 

naps2.console -o "F: \ recibo digitalizado.pdf" -v

--progress
Exibe uma janela gráfica para o progresso da digitalização. 

naps2.console -o "F: \ scanned receipt.pdf" --progress

-n, --number {#}
(Padrão: 1) O número de varreduras a serem executadas. Isso pode ser diferente do número de páginas digitalizadas (por exemplo, se você tiver um alimentador automático de documentos). 

naps2.console -o "F: \ recibos digitalizados.pdf" -n 5

-d, --delay {#}
(Padrão: 0) O atraso (em milissegundos) entre cada varredura. 

naps2.console -o "F: \ recibos digitalizados.pdf" -n 5 -d 5000

-f, --force
Sobrescreve os arquivos existentes. Se não for especificado, quaisquer arquivos que já existam não serão alterados (e uma mensagem de erro será exibida). 

naps2.console -o "F: \ recibo digitalizado.pdf" -f

-w, --wait
Após terminar, aguarde a entrada do usuário (enter / return) antes de sair. 

naps2.console -o "F: \ recibo digitalizado.pdf" -w

--install {component}
Instala o componente especificado. Valores possíveis: "genericimport", "ocr-eng" (e outros códigos de idioma - ver referência )

--help
Exibe uma tela de ajuda similar a esta página da wiki. 

naps2.console --help

Espaços reservados

Quaisquer argumentos de caminho (-o, -e, etc.), bem como alguns argumentos de texto (--subject, --body) podem usar marcadores de posição. Os seguintes marcadores de posição estão disponíveis:

$(YYYY)- o ano em curso.
$(YY)- O ano atual (00-99).
$(MM)- O mês atual (01-12).
$(DD)- O dia atual (01-31).
$(hh)- A hora atual (00-23).
$(mm)- O minuto atual (00-59).
$(ss)- O segundo atual (00-59).
$(nnnn)- Um número incremental único (0001, 0002, etc.).
$(nnn)- Um número incremental único (001, 002, etc.).
$(nn)- Um número incremental único (01, 02, etc.).
$(n)- Um número incremental único (1, 2, etc.).
